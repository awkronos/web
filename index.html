Motivation

Agent data portability

AI agents accumulate context about users—preferences, interaction history, learned behaviors. This data is typically stored in vendor-specific formats within proprietary infrastructure. Users cannot migrate this context when switching providers, audit what their agent has learned, or share context selectively with other agents.

This specification addresses the data storage layer, complementing the W3C AI Agent Protocol Community Group's work on agent identity, discovery, and communication.

Design Goals

Specification objectives

Portability

Enable users to migrate between agent implementations without losing accumulated context. Agents read a common format from user-controlled storage.

Privacy

Agent data is private by default. Users grant explicit, granular permissions. Encryption at rest is specified for sensitive containers.

Interoperability

Built on Solid Protocol, RDF, and established W3C standards. Vocabulary extends FOAF and PROV-O for maximum compatibility.
